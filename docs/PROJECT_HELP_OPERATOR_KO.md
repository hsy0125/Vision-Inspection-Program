# InspectionProgram 작업자용 도움말

이 문서는 현장에서 검사 기능을 사용하는 작업자용 가이드입니다.

## 1) 작업 시작 전 확인

- 상단 모드를 `Teaching` 또는 `Auto Run`으로 맞춥니다.
- 상단 언어(`Kr / En / Cn / Jp / Vn / In`)를 필요한 언어로 선택합니다.
- 검사 대상 이미지 파일 또는 폴더를 준비합니다.

## 2) 기본 작업 순서

1. `Teaching` 이동
2. `Load`로 기준 이미지 로드
3. `ROI+`로 검사 영역 지정
4. 필요 시 `NCC 모델 저장`
5. `검사 레시피 저장`
6. `Auto Run` 이동
7. `Load`로 검사할 이미지(단일/폴더) 선택
8. `3) Run inspect` 실행 (또는 상단 `개수` 메뉴)
9. 결과 확인 후 `4) CSV 저장`
10. 필요 시 상단 `로그`로 Inspection Log를 `.txt` 저장

## 3) 검사 UI 흐름

### 1. Load
- [예]: 폴더 전체 이미지 목록 로드
- [아니오]: 이미지 1장 로드

### 2. Threshold
- 밝기 하한(이진화) 미리보기 조정

### 3. Run
- 티칭에서 저장한 레시피를 적용 후 검사
- NCC 모델이 있으면 전역 패턴 카운트
- NCC 모델이 없으면 ROI 기반 Blob 검사
- `자동 사이클` 체크 시 폴더 전체 순차 검사

### 4. CSV 저장
- 세션 검사 결과(OK/NG) 저장

## 4) 자주 쓰는 버튼

### Teaching
- `Load`, `Save`, `Clear`
- `ROI+`, `ROI Save`
- `NCC 모델 저장`, `NCC 검사`, `카운트`
- `검사 레시피 저장`

### Auto Run
- `Load`, `Clear`, `Save`
- `3) Run inspect`, `Stop`, `4) CSV 저장`
- 상단 `개수`, `검사`, `로그`

## 5) 결과 해석

- `OK`: 정상 판정
- `NG`: 비정상 판정
  - 예: ROI 없음, Blob 0개, 기대 개수 불일치, NCC 매칭 실패

## 6) 문제 발생 시 체크

- Run 눌렀는데 동작 안 함
  - 이미지가 로드되었는지 확인
- NG가 계속 발생
  - ROI 위치/크기 재설정 후 레시피 재저장
- NCC 기반 결과가 이상함
  - NCC 모델 재생성 후 다시 저장
- 자동 사이클이 안 됨
  - `Load`에서 폴더 모드로 열었는지 확인

## 7) 저장 파일

- 로그 텍스트: `.txt`
- 검사 결과: `.csv`
- 저장 위치는 저장 대화상자에서 지정
